SS05-01

Display Counters Summary

This service allows the User to view counters summary for the assigned Administration Tasks once the User clicks on the 'Administration Tasks /الطلبات الإدارية' Tab Page.

1- Login as "Inspector" "المفتش"

1- The User clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays the "My Tasks/مهماتي" page and fetches all related counters data from DARB Back-End System:

    3.1- Administration Tasks: The System Displays the number for the new assigned tasks in red star.

    3.2- New Tasks: The System Displays the number for the new assigned tasks.

    3.3- Tasks in Progress: The System Displays the number of inspection tasks in progress (not closed yet).

    3.4- Accomplished Tasks: The System Displays the number of Accomplished Tasks (closed).

    3.5- Delayed Tasks: The System Displays the number of delayed tasks (tasks that are not closed within 24 hours since the Assignment Date & Time.

SS05-04

View Trading License Information

1- Login as "Inspector" "المفتش"

1- The User clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays the "My Tasks/مهماتي" page and fetches all assigned inspection tasks for the current logged-in User  from DARB Back-End System.

SS05-05

Start Inspection Task

This service allows the User to start the inspection task for the selected Establishment.

1- Login as "Inspector" "المفتش"

1- The User clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays the "My Tasks/مهماتي" page and fetches all assigned inspection tasks for the current logged-in User  from DARB Back-End System.

4- The User clicks on "Start Inspection/بدء المهمة" button for the selected Establishment in the list.

5- The System redirects the page view to display the "Establishment Inspection/تسجيل زيارة منشأة" and the inspector proceeds with the inspection process [Goto: SCR-04 Establishment Inspection SAD].

    5.1 The System captures the current System Date to fill Inspection Start Date field.

    5.2 The System captures the current System Time to fill Inspection Start Time field.

    5.3 The System updates the Task Status from 'New' to 'Inspection in Progess'.

1. GPS Location Service must be enabled in the Smart Device.

 

2. The User must be within 50 meters of The perimeter of the Establishment, if not, the System displays an alert message for the User and don't allow the User to start the inspection task.

SS05-06

Trasfer Inspection Task (Delegate)

This service allows the User to transfer (delegate) an inspection task for the selected Establishment to another inspector.

1- Login as "Inspector" "المفتش"

1- The User clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays the "My Tasks/مهماتي" page and fetches all assigned inspection tasks for the current logged-in User  from DARB Back-End System.

4- The User clicks on "Transfer Task/تحويل المهمة" button for the selected Establishment in the list.

5- The System opens a popup page to allow the inspector to select the alternative inspector:

    5.1 The User selects the Alternative Inspector from the dropdown list.

    5.2 The User enters the Reason(s) for Transferring the Task.

    5.3 The User clicks on Submit button.

    5.4 The System sends an Email Notification to the selected Alternative Inspector (with the Reason(s) for Transferring the Task) in order to Accept or Reject the transferred task (the Inspectors' Supervisor is included in the Email Notification in the CC Section).

    5.5 The System sends a PUSH Notification to the selected Alternative Inspector Smart Device to indicate there is a transferred inspection task.

    5.6 The System hides this transferred task from the Transferor (who initiate the transfer request) and the task become visible to the Transferred To.

1. The Inspectors List and their Email IDs are fetched from the Active Directory of RAK DED (Integration Point).

SS05-07

Postpone Inspection Task

This service allows the User to an inspection task for the selected Establishment to a new calendar date.

1- Login as "Inspector" "المفتش"

1- The User clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays the "My Tasks/مهماتي" page and fetches all assigned inspection tasks for the current logged-in User  from DARB Back-End System.

4- The User clicks on "Postpone Task/تأجيل المهمة" button for the selected Establishment in the list.

5- The System opens a popup page to allow the inspector to select a new inspection date for the selected task:

    5.1 The User selects the New Inspection Date from the calendar control.

    5.2 The User enters the Reason(s) for Postponing the Task.

    5.3 The User clicks on Submit button.

    5.4 The System updates the Inspection Task Date to the New Inspection Date. 

    5.5 The System sends an Email Notification to the Inspector's Supervisor about the Postponed Task with the Reason(s) for Postponing the Task.

    5.6 The System sends an Email & SMS Notification to the Establishment Owner about the Postponed Task.

    5.7 The System moves the postponed task from the Daily Task Tab Page to the Agenda Tab Page since it is postponsed into a future date.

1. The Inspector Supervisor Email ID is fetched from the Active Directory of RAK DED.

2. The Establishment Owner Email ID & Establishment Owner Mobile Number are fetched from SAP Trading License Back-End System (Integration Point).

SS05-08

Accepting Trasnferred Task

This service allows the User to Accept a transferred inspection task.

1- Login as "Inspector" "المفتش"

1- The User received the PUSH Notification about the transferred task, opens the Smart Device, and clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays the "My Tasks/مهماتي" page and fetches all assigned inspection tasks (& transferred tasks) for the current logged-in User  from DARB Back-End System.

4- The User clicks on "Accept Task/قبول المهمة" button for the selected Establishment in the list.

5- The System sends an Email Notification to the selected Inspector Transferor (the Inspectors' Supervisor is included in the Email Notification in the CC Section) that the transferred task is accepted.

6- The System hides & shows the following buttons after the is accepted:

    6.1- Reject button (hide).

    6.2- Accept button (hide).

    6.3- Start Inspection Task (show).

    6.4- Transfer Inspection Task (show).

    6.5- Postpone Inspection Task (show).

1. The User cannot do any other action buttons on the transferred task before Accepting the transferred task.

SS05-09

Rejecting Transferred Task

This service allows the User to Reject a transferred inspection task.

1- Login as "Inspector" "المفتش"

1- The User received the PUSH Notification about the transferred task, opens the Smart Device, and clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays the "My Tasks/مهماتي" page and fetches all assigned inspection tasks (& transferred tasks) for the current logged-in User  from DARB Back-End System.

4- The User clicks on "Reject Task/رفض المهمة" button for the selected Establishment in the list.

5- The System opens a popup page to allow the inspector to enter the Reason(s) for Rejecting the Transferred Task:

    5.1- The User enters the Reason(s) for Transferring the Task.

    5.2- The User clicks on Submit button.

    5.3- The System sends an Email Notification to the selected Inspector Transferor (the Inspectors' Supervisor is included in the Email Notification in the CC Section) that the transferred task is rejected.

6- The System hides this transferred task and make it visible to the Transferor (who initiate the transfer request).

1. The User cannot do any other action buttons on the transferred task before Rejecting the transferred task.

SS05-10

Initiator List

Displays a list of initiators who initiated the Administration Task.

 

Based on User's selection the Tabel object filters the List of Assigned Tasks on the right-side.

1- Login as "Inspector" "المفتش"

1- The User clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The User clicks on "Administration Tasks/الطلبات الإدارية" Tab Page.

3- The System displays and fetches all future assigned inspection tasks for the current System date for the current logged-in User  from DARB Back-End System.

4- Based on User's clicks on any row in the Initiator Tabel, the tabel filters the List of Assigned Tasks on the right-side.
