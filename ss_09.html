SS09-01

Inspection Basic Information

This service displays the Basic information about the current Establishment Inspection Session & allows the User to enter final decision about the inspected establishment.

1- Login as "Inspector" "المفتش"

1- The User clicks on "My Tasks/مهماتي" button (in the Landing Page)

2- The System displays the "My Tasks/مهماتي" page and fetches all assigned inspection tasks for the current logged-in User  from DARB Back-End System.

3- The User clicks on "Start Inspection/بدء المهمة" button for the selected Establishment in the list.

4- The System redirects the page view to display the "Establishment Inspection/تسجيل زيارة منشأة" and the inspector proceeds with the inspection process [Goto: SCR-09 Establishment Inspection SAD].

    4.1 The System captures the current System Date to fill Inspection Start Date field.

    4.2 The System captures the current System Time to fill Inspection Start Time field.

    4.3 The System updates the Task Status from 'New' to 'Inspection in Progess'.

    4.4 The System gets & displays Tasks' Summary information as described in the mock-up.

5- The User selects the "حالة المنشأة" from the dropdown list.

6- The User selects the "قرار المفتش" from the dropdown list at the end of inspection session.

7- In case the inspection visit needs to be postponed or rescheduled, the User can select a new Inspection Date "تأجيل الزيارة" from the calendar control.

8- (if required) The User takes captures photos about the inspected establishment using the buil-in camera in the smart device, and these photos are saved in the device local memory, as long as the inspection session is opened, and it will be transfered to DARB Back-End System once the inspector ends the inspection session.

9- The User takes the signature of the establishment Owner or Manager. In case the Owner or the Manager refused to sign, the User checks the option "صاحب المنشأة/المدير رفض التوقيع".

10- The User enters his notes "ملاحظات المفتش" (if any) regarding the currect inspection visit.

11- Once the User is done with the inspection session (after doing the Checklists, Corrective Actions, Violations (if any), Confiscated Goods (if any), Site Survey (if any), the User ends the visit using the "إنهاء الزيارة" button.

12- The System collects all inspection session data and submit it to DARB Back-End System to be saved.

13- DARB Back-End System generates the Inspection Visit Report (in PDF Format) and send it to the Establishment Owner/Manager Emails.

14- In case, the inspection task is about "Site Survey", then DARB Back-End System generates the Site Survey Report (in PDF Format) and send it to the Establishment Owner/Manager Emails.

15- DARB Back-End System sends SMS Notification to the Establishment Owner/Manager mobile number, that the inspectoin session is over.

1. The Establishment Owner Email ID & Establishment Owner Mobile Number are fetched from SAP Trading License Back-End System (Integration Point).

SS09-02

Inspection Historical Visits

This service displays the Historical Visits for the current inspected establishment.

1- Login as "Inspector" "المفتش"

1- The User clicks on "Inspection Historical Visits/الزيارات السابقة للمنشأة" button inside "Establishment Inspection/تسجيل زيارة منشأة" page.

2- The System displays a tabel contains the following columns:

    2.1 Index.

     2.2 Inspection Date (Visit Date)

     2.3 Inspector Name.

     2.4 Attachments (Inspection Report, Confiscated Goods Report, Undertaking Report: this report is retrieved from SAP Trading License Back-End System)

3- The User can view any of the provided attachments (if required).

SS09-03

Inspection Checklists

This service displays the Inspection Checklist Items for the current inspected establishment.

1- Login as "Inspector" "المفتش"

1- The User clicks on "Inspection Checklist/قائمة التحقق" button inside "Establishment Inspection/تسجيل زيارة منشأة" page.

2- The System displays a list of checklist questions based on the Establishment's Business Activity.

3- The User answers each question either (Yes or No) during the establishment inspection session.

SS09-04

Inspection Corrective Actions

This service displays the Inspection Corrective Action Items (based on the provided answers in the Previous step (SS09-03: Inspection Checklist)) for the current inspected establishment.

1- Login as "Inspector" "المفتش"

1- The User clicks on "Inspection Corrective Actions/الإجراءات التصحيحية" button inside "Establishment Inspection/تسجيل زيارة منشأة" page.

2- The System displays a list of corrective actions based on the provided answers in the Previous step (Inspection Checklist) & the Total Number of Occurence for each corrective action for inspected establishment.

3- The User selects one or more corrective actions (based on the inspector judgment).

SS09-05

Inspection Violations

This service allows the User to register one or more violations during the Inspection Session for the current visited establishment.

 

This functionality is available & visible to all Inspection Task Types except "Campaign Awearness Inspection Task Type" الحملات التوعوية لا يوجد بها مخالفات، فقط يتم توعية صاحب المنشأة بالإجراءات التصحيحية.

1- Login as "Inspector" "المفتش"

1- The User clicks on "Inspection Violations/المخالفات" button inside "Establishment Inspection/تسجيل زيارة منشأة" page.

2- The System displays a list of the registered violations for the current inspection visit (session).

3- The User can Edit/Delete any of the registered violations as long as the inspection visit is not closed.

4- The User clicks "إضافة مخالفة جديدة" button to add new violation for the current inspection session.

5- The System opens a new page to add the new violation data.

6- The User fills the new violation information (check CTG_09.05: Inspection Violations Control) & clicks "Save" button to go back to the "Inspection Violations/المخالفات" page.

7- DARB Back-End System generates one or more new inspection task based on the "Violation Due Date/تاريخ الإستحقاق", Once the Inspection Session is closed by the inspector and the data is sent back to DARB Back-End System. These new generated tasks are named "Violations Follow-up Task/مهمة متابعة المخالفات"

SS09-06

Inspection Confiscated Goods

This service allows the User to confiscate one or more goods during the Inspection Session for the current visited establishment.

 

This functionality is available & visible to all Inspection Task Types except "Campaign Awearness Inspection Task Type" الحملات التوعوية لا يوجد بها حجز مواد، فقط يتم توعية صاحب المنشأة بالإجراءات التصحيحية.

1- Login as "Inspector" "المفتش"

1- The User clicks on "Inspection Confiscated Goods/المواد المحجوزة" button inside "Establishment Inspection/تسجيل زيارة منشأة" page.

2- The System displays a list of the confiscated items for the current inspection visit (session).

3- The User can Edit/Delete any of the confiscated items as long as the inspection visit is not closed.

4- The User clicks "إضافة مادة جديدة" button to add new confiscated item for the current inspection session.

5- The System opens a new page to add the new confiscated item data.

6- The User fills the new confiscated item information (check CTG_09.06: Inspection Confiscated Goods Control) & clicks "Save" button to go back to the "Inspection Confiscated Goods/المواد المحجوزة" page.

7- Once the Inspection Session is closed by the inspector and the data is sent back to DARB Back-End System to analyze these confiscated items and to take do furthure actions on the analysis results.

SS09-07

Inspection Appeals

This service lists any (In Progress) submitted appeals for the current inspected establishment.

 

This is an informative list for the inspector to avoid registering any violations for the current establishment as long as there is any in progress appeals.

 

This functionality is available & visible to all Inspection Task Types except "Campaign Awearness Inspection Task Type" الحملات التوعوية لا يوجد بها تظلمات، فقط يتم توعية صاحب المنشأة بالإجراءات التصحيحية

1- Login as "Inspector" "المفتش"

1- The User clicks on "Inspection Appeals/التظلمات" button inside "Establishment Inspection/تسجيل زيارة منشأة" page.

2- The System displays a tabel contains the following columns:

    2.1 Appeal Date.

     2.2 Submitted By.

     2.3 Violation Number.

     2.4 Violation Main Type.

    2.5 Violation Sub Type.

    2.6 Appeal Status.

1- This list is returned through an integration point with RAK DED Appeals System.

SS09-08

Inspection Site Survey

This service allows the User to Site Survey the current visited establishment based on pre-scheduled appointment set in the Appointment Module of RAK DED Website (Integration Point).

 

This functionality is ONLY available & visible to Inspection Task Type "Site Survey Request طلب معاينة موقع" and some other "Work Permits طلب تصريح".

1- Login as "Inspector" "المفتش"

1- The User clicks on "Inspection Site Survey/معاينة الموقع" button inside "Establishment Inspection/تسجيل زيارة منشأة" page.

2- The System displays the Site Survey page.

3- The User fills the Site Survey Information (check CTG_09.08: Inspection Site Survey) & clicks "Save" button to save Site Survey Information in the local memory of the smart device.

4- Once the User ends the visit using the "إنهاء الزيارة" button, DARB Back-End System generates the Site Survey Report (in PDF Format) and send it to the Establishment Owner/Manager Emails.

1- Appointment Data is retreived through an integration point with the RAK DED Appointment System.
